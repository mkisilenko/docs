## Database Options

Choose the approach that fits your needs:

<Tabs>
  <Tab title="Anyx Managed (Easiest)">
    ### We Handle Everything

    **What it is:**

    - Anyx provisions a Supabase database for you
    - Zero configuration needed
    - Fully managed and maintained
    - Auto-migrations when you add features

    **Best for:**

    - Beginners
    - Quick prototypes
    - You don't want to manage infrastructure
    - You want to focus on building

    **How it works:**

    1. Mention "with database" in your prompt
    2. Anyx provisions Supabase project (2-3 min)
    3. Database credentials auto-configured
    4. Start using immediately

    **What you get:**

    - Postgres database
    - Auto-scaling
    - Daily backups
    - Unlimited API requests

    <Check>
      Best choice for 90% of users
    </Check>
  </Tab>
  <Tab title="Bring Your Own Supabase">
    ### Connect Your Database

    **What it is:**

    - Use your existing Supabase project
    - You control the database
    - Anyx handles migrations (auto or manual)
    - Full ownership

    **Best for:**

    - You already have a Supabase project
    - Production apps
    - You want full database control
    - Team collaboration on database

    **How to connect:**

    1. Go to project settings in Anyx
    2. Click "Connect Database"
    3. Choose "Bring Your Own Supabase"
    4. Enter Supabase URL and anon key
    5. Choose migration mode:
       - **Automated**: Anyx applies migrations
       - **Manual**: You review and paste SQL

    **Migration modes:**

    **Automated** (recommended):

    - Anyx applies database changes automatically
    - Faster iterations
    - Safe rollback on errors

    **Manual** (safest):

    - Anyx generates SQL for you
    - You review and paste in Supabase dashboard
    - Full control over every change

    <Tip>
      Start with manual, switch to auto once comfortable
    </Tip>
  </Tab>
  <Tab title="No Database">
    ### Frontend Only

    **What it is:**

    - No backend or data storage
    - Static frontend app
    - Fastest build time

    **Best for:**

    - Landing pages
    - Portfolios
    - Marketing sites
    - Pure frontend demos
    - Static content sites

    **How to specify:** Just don't mention database in your prompt, or say:

    ```
    "Create a landing page (no database needed)"
    ```

    **What you get:**

    - React frontend
    - Deployed to production
    - CDN-delivered (fast globally)
    - Scales infinitely

    <Check>
      Deploys in 5-8 minutes (fastest option)
    </Check>
  </Tab>
</Tabs>

## Deployment & Hosting

Every app is automatically deployed to production:

### What Happens

<Steps>
  <Step title="GitHub Repository Created">
    - Anyx creates repo in your GitHub account
    - You own the code (not vendor lock-in)
    - Full git history
    - Can clone locally anytime

    **Repo includes:**

    - Source code
    - Tests
    - GitHub Actions (CI/CD)
    - Documentation
  </Step>
  <Step title="Production Deployment">
    - Repo connected to hosting automatically
    - Production URL generated
    - SSL certificate (HTTPS)
    - Global CDN

    **You get:**

    - `https://your-app.anyx.app`
    - Or connect custom domain
    - Automatic deployments on changes
  </Step>
  <Step title="Environment Variables Set">
    All API keys and secrets configured:

    - Database credentials
    - API keys (if needed)
    - Feature flags

    **Secure:**

    - Encrypted at rest
    - Never in source code
    - Environment-specific
  </Step>
</Steps>

### Custom Domains

Connect your own domain in project settings:

1. Add custom domain
2. Update DNS records
3. SSL auto-configured

**Supported:**

- `yourapp.com`
- `app.yourcompany.com`
- Multiple domains
