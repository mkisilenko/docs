## Migration Process

<Steps>
<Step title="Connect GitHub">
  1. Sign in to Anyx with your GitHub account
  2. Grant repository access
  3. Anyx detects your Lovable projects automatically
  
  <Tip>Anyx recognizes Lovable projects via package.json signatures</Tip>
</Step>

<Step title="Select Project">
  1. Choose which Lovable project to migrate
  2. Anyx analyzes:
     - Database migrations (if any)
     - Supabase edge functions
     - Dependencies
     - Project complexity
  
  **Analysis takes:** 30-60 seconds
  
  <Frame>
    <img src="/images/checks-passed.png" alt="Project analysis screen" />
  </Frame>
</Step>

<Step title="Choose Database Option">
  Select how to handle your database:
  
  <Tabs>
  <Tab title="Keep Your Supabase">
    **If you already have Supabase connected:**
    
    1. Anyx connects to your existing Supabase
    2. Choose migration mode:
       - **Auto**: Anyx applies migrations
       - **Manual**: You paste SQL (like Lovable)
    3. All data preserved
    
    <Check>Recommended for production projects</Check>
  </Tab>
  
  <Tab title="Switch to Anyx Managed">
    **If you want easier database management:**
    
    1. Anyx provisions new Supabase
    2. Optionally migrate data from old database
    3. Fully managed going forward
    
    <Check>Easiest option - zero maintenance</Check>
  </Tab>
  
  <Tab title="No Database">
    **If your project is frontend-only:**
    
    1. No database setup needed
    2. Fastest migration
    3. Deploy immediately
    
    <Check>Perfect for landing pages, portfolios</Check>
  </Tab>
  </Tabs>
</Step>

<Step title="Complete Migration">
  Anyx finalizes the setup:
  
  - ✅ Imports your codebase
  - ✅ Connects database (if chosen)
  - ✅ Applies migrations (if needed)
  - ✅ Sets up production deployment
  - ✅ Configures environment variables
  - ✅ Runs initial tests
  
  **Time:** 2-3 minutes
  
  <Check>Your project is now ready to continue building in Anyx!</Check>
</Step>
</Steps>

## Common Questions

<AccordionGroup>
<Accordion title="Will my app stay live during migration?">
  **Yes!** Migration is non-destructive:
  - Your current deployment stays active
  - Anyx sets up in parallel
  - You control when to switch
  - Can test before going live
</Accordion>

<Accordion title="What if I have custom code?">
  **All custom code is preserved:**
  - Custom components kept
  - Manual code edits maintained
  - npm dependencies unchanged
  - Build scripts work as before
  
  Anyx works with your customizations
</Accordion>

<Accordion title="Can I migrate back to Lovable?">
  **Yes!** Your code stays in your GitHub:
  - You own the repository
  - Can point Lovable back to it
  - No vendor lock-in
  - Export anytime
  
  But you probably won't want to :)
</Accordion>

<Accordion title="What happens to my deployment?">
  **Two options:**
  1. **Maintain existing:** Keep your current hosting setup
  2. **Fresh deploy:** Anyx sets up new production environment
  
  Your custom domain can be transferred easily
</Accordion>

<Accordion title="Is my data safe?">
  **Absolutely:**
  - No data deleted
  - Optional data migration (you control)
  - Encrypted transfers
  - Rollback available
  
  We never access your data without permission
</Accordion>

<Accordion title="How long does migration take?">
  **Typical times:**
  - Simple site: 2-3 minutes
  - App with database: 4-5 minutes
  - Complex multi-page: 5-7 minutes
  
  Most of this is automated analysis and setup
</Accordion>
</AccordionGroup>
